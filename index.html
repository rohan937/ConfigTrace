The blind spot
Production incidents don't start in Git.

Code changes go through pull requests, code review, and CI/CD. But configuration changes bypass all of that. They happen in dashboards, CLIs, and vendor portals — with no history, no diff, and no way to trace them after the fact.

The full flow

Snapshot. Diff. Risk-assess. Repeat.

ConfigTrace treats your live infrastructure like a repository. Every sync is a commit. Every change is reviewable. Manual Sync Now is always available alongside the hourly schedule.

01 · CONNECT

Connect a provider

Add a scoped API token and a zone or account ID. Credentials are encrypted before storage and never shown again.

02 · BASELINE

Create a baseline

The first sync snapshots your current configuration — every DNS record, rule, and field value — as your known-good baseline.

03 · SYNC

Sync hourly or manually

ConfigTrace syncs your zone every hour. Hit Sync Now anytime to check for changes on demand, without waiting for the schedule.

04 · DIFF

Detect field-level changes

Every sync is diffed against the previous state. Exactly what changed surfaces: the old value, the new value, and when.

05 · CLASSIFY

Classify change risk

Each change is scored low to critical based on blast radius. A changed TXT comment is not the same as a rerouted CNAME.

06 · REVIEW

Review the timeline

A clear change timeline shows what happened, when, and what it affected — so "what changed?" gets answered in seconds.

Cloudflare DNS · Live

Connect a Cloudflare zone with a scoped API token and Zone ID. ConfigTrace monitors every DNS record continuously. Hourly syncs run automatically; manual syncs are always available. The first sync creates your baseline — every sync after diffs against it.

A / AAAA
IPv4 and IPv6 address records — the most traffic-critical changes
CNAME
Canonical name aliases — rerouting these can redirect live traffic
MX
Mail exchange records — changes here can silently break email delivery
TXT
SPF, DKIM, DMARC, domain ownership, and other text records
NS
Name server records — changes here affect your entire zone
TTL
Time-to-live values — unexpectedly short TTLs can signal impending changes
Proxied
Cloudflare proxy on/off — switching this changes your security and routing model
Added / Removed
New records and deleted records are both tracked and surfaced as changes

Risk classification

Not all changes are equal.

ConfigTrace weighs each detected change by its potential blast radius so your team can triage at a glance — not after the incident is already in progress.

Low
TXT comment changed
Cosmetic or metadata change. No routing or security impact expected.
Medium
New subdomain added
TTL shortened
Expanded attack surface or cache behavior change. Worth a closer look.
High
API CNAME rerouted
Production traffic may be flowing somewhere it shouldn't be.
Critical
Root A record deleted
NS record changed
The site may be offline or your entire zone could be affected.
